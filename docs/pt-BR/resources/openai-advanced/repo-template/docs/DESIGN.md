# DESIGN.md

Este arquivo é o ponto de entrada do design. Mantenha-o breve e use-o para direcionar aos arquivos mais detalhados em `docs/design-docs/`.

## Propósito

Registrar decisões duráveis de design de produto e de sistema que devem sobreviver além de um único chat, sprint ou da memória de um revisor.

## Leia Isto Quando

- você precisar da filosofia de design atual.
- você estiver prestes a introduzir um novo padrão.
- você precisar saber quais decisões de design estão resolvidas versus quais ainda estão em aberto.

## Documentos de Design Canônicos

- `docs/design-docs/index.md`: índice de documentos aceitos, propostos e descontinuados.
- `docs/design-docs/core-beliefs.md`: crenças fundamentais do projeto focadas em agentes.

## Regras de Design

- Mantenha os documentos de design pequenos e atualizados.
- Prefira um documento por área de decisão.
- Vincule os documentos de design a partir de planos e especificações quando uma mudança depender deles.
- Se uma regra de design se tornar operacionalmente crítica, promova-a para uma verificação automatizada ou atualize o arquivo `ARCHITECTURE.md`.
