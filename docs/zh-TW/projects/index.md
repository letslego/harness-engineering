# 歡迎來到專案實戰

這裡是 Learn Harness Engineering 的動手實踐部分。僅僅閱讀講義是不夠的，你需要親自搭建環境，觀察 Codex 或 Claude Code 等代理在不同規則下的真實表現。

## 專案概覽

本課程包含 6 個漸進式的實戰專案，帶你從零開始建構完整的代理工作環境：

1. **提示詞驅動 vs 規則驅動**：對比純提示詞驅動的代理和帶有基礎 harness 的代理在表現上的差異。
2. **讓專案可讀並接住上次工作**：學習如何重構儲存庫結構，使其對 AI 更加友好，並建立工作交接機制。
3. **跨工作階段工作連續性**：設計狀態檔案和初始化腳本，讓代理可以在多次對話中斷後無縫恢復工作。
4. **執行回饋與範圍控制**：引入執行時期回饋機制，讓代理能夠自行檢查程式碼是否能正常執行，並修正錯誤。
5. **自我驗證與角色分離**：建立獨立的評審與驗證環節，防止代理產生「幻覺」或提前宣告完成。
6. **綜合代理工作環境**：搭建一套完整且具備可觀測性的代理工作環境（綜合演練）。

## 如何進行

每個專案資料夾中通常包含：
- `starter/`：你的起始工作區。
- `solution/`：參考的最終實作（如果你卡住了可以參考）。
- 相關的指導檔案，說明本次任務的具體背景和目標。

請使用你習慣的 AI 程式設計代理（如 Claude Code、Cursor、Trae 等）在每個專案的 `starter/` 目錄下完成任務。
