# PRODUCT_SENSE.md

Este arquivo captura o julgamento durável do produto que os agentes não conseguem inferir de forma confiável apenas a partir do código.

## Essência do Produto (Product Core)

- Usuário principal: `[substitua]`
- Trabalho a ser feito (Job to be done): `[substitua]`

## Regras do Produto

- Favoreça a confiabilidade visível ao usuário em detrimento da contagem de recursos.
- Trate comportamentos ambíguos como uma lacuna na especificação, não como permissão para adivinhar.
- Se a implementação alterar o que os usuários veem ou em que confiam, atualize a especificação correspondente.
- Use especificações de produto para fluxos concretos e use este arquivo para prioridades transversais do produto.

## Padrões Proibidos (No-Go Patterns)

- Ações destrutivas ocultas.
- Falha silenciosa sem feedback ao usuário.
- Fonte da verdade obscura para o estado visível.
- Recursos que não podem ser explicados em uma única frase.
