## 產品規則

- 優先處理使用者可見的可靠性，再考慮功能數量。
- 如果行為有歧義，就把它視為 spec 缺口，不要據此猜測。
- 如果實作改變了使用者看見或信任的內容，就更新對應的 spec。
- 具體流程寫在 product spec；跨流程的產品優先順序寫在這份檔案。

## 禁區模式

- 隱藏的破壞性操作
