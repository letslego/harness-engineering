# PLANS.md

這份檔案定義執行計畫如何建立、更新、完成與歸檔。

## 什麼時候必須有 plan

當工作符合以下任一條件時，就建立 execution plan：

- 會跨越多個工作階段
- 會變更多個子系統
- 存在不可忽略的驗證或上線風險
- 依賴應該記錄下來的開放決策

## 計畫放哪

- `docs/exec-plans/active/`：目前正在推動工作的計畫
- `docs/exec-plans/completed/`：已完成，但為了後續代理脈絡而保留的計畫
- `docs/exec-plans/tech-debt-tracker.md`：延後處理的工作與 follow-up

## 最少要包含的部分

- 目標
- 範圍與明確排除的項目
- 驗證路徑
- 風險與 blocker
- 進度日誌
- 開放決策

## 執行規則

- 每一份 active plan 在同一時間都應該只有一個明確負責的目前步驟。
- plan 要隨著工作推進而更新，不要把它視為靜態說明文件。
- 如果某個決策改變了實作方向，就把它記到 plan 裡。
- 已完成的計畫要移到 `completed/`，讓後續代理仍然能找到先前的脈絡。
