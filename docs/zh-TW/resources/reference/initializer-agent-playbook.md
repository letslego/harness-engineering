# 初始化代理操作手冊

在儲存庫中的第一個重要工作階段使用這份手冊，也就是增量功能開發開始前的階段。

## 目標

建立穩定的操作面，讓後續工作階段可以直接實作行為，不必重新推導啟動命令、目前狀態或任務邊界。

## 必要產出

初始化代理至少應留下這些工件：

- 根指令檔，例如 `AGENTS.md` 或 `CLAUDE.md`
- 機器可讀的功能面，例如 `feature_list.json`
- 持久的進度工件，例如 `claude-progress.md`
- 標準的啟動輔助工具，例如 `init.sh`
- 記錄基礎腳手架狀態的初始安全提交

## 檢查清單

1. 定義標準啟動路徑。
2. 定義標準驗證路徑。
3. 建立進度日誌並記錄起始狀態。
4. 把工作拆成具名功能並標註狀態。
5. 建立第一個乾淨的基準提交。

## 成功測試

一個沒有任何先前聊天脈絡的新工作階段，應該能回答：

- 這個儲存庫的用途
- 如何啟動它
- 如何驗證它
- 哪些工作尚未完成
- 下一個最佳步驟是什麼
