# FRONTEND.md

這份檔案定義穩定的前端預期，避免代理不可預測地建立不同的 UI 模式。

## UI 原則

- 先以易懂為優先，再考慮新穎性。
- 互動流程要容易被發現，並且可以重新開始。
- 優先使用少量可重複使用的元件，避免建立一次性的變體。
- 無障礙檢查屬於一般驗證的一部分，也要納入日常工作流程。

## 護欄

- 在 `docs/references/` 記錄設計系統或元件函式庫。
- 記錄關鍵的使用者可見狀態，包含 empty、loading、success、error、retry。
- 在不同流程中維持文案、鍵盤行為與視覺層級一致。
- 修復 UI bug 後，要新增或更新對應的驗證步驟。

## 驗證要求

- 為關鍵使用者旅程保留證據。
- 把瀏覽器或執行階段的驗證步驟寫進相關 plan。
- 如果視覺迴歸經常發生，就標準化截圖或 DOM 檢查。
